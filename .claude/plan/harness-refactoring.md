# Implementation Plan: Harness Refactoring

## Design Principles Applied

1. **Simplification**: Fewer files, not more. Don't split patterns.md — all 6 agents read it.
2. **Single source of truth**: Remove duplicated guidance; don't create new reference files.
3. **Security by design**: Already done. No refactoring needed.
4. **Core is domain-blind**: Already clean. No refactoring needed.

## Key Insight

The biggest refactoring win is NOT splitting large files — it's **removing duplication between files**. Content that exists in both SKILL.md and coordinator.md should exist in exactly one place.

## What to Refactor

### R1: Remove duplicated guidance from coordinator.md (253 → ~170 lines)

coordinator.md duplicates content from SKILL.md and patterns.md:
- **Script Calls section** (38 lines) — full subcommand examples. Replace with: "Run `node plugins/harness/scripts/harness-companion.mjs --help` for all subcommands."
- **Calibration Enforcement** (7 lines) — paraphrases SKILL.md. Replace with one-line reference.
- **Sprint Retrospective** (5 lines) — paraphrases SKILL.md. Replace with one-line reference.
- **Pause Rules** (content overlaps SKILL.md stop conditions) — keep only coordinator-specific rules.

Estimated: -83 lines. No content lost — it still exists in SKILL.md or --help.

### R2: Trim patterns.md templates that are never used (702 → ~600 lines)

Check which templates are actually generated by agents vs just sitting in patterns.md as dead examples:
- `cost-log.md` template — never generated. The cost_tracking lives in state.json. Remove.
- `decomposition.md` template — was removed in v3.0.0 but template may still be in patterns.md.
- `init.sh` / `init.bat` templates — initializer generates these fresh each time without reading the template. If true, remove templates.

Estimated: -50 to -100 lines of dead templates.

### R3: Clean harness-companion.mjs (314 → ~280 lines)

- Move `summarizeMetricsInline()` (40 lines) inside the `postmortem-data` case — it's only called there
- Remove historical comments ("replaces former lib/metrics.mjs", "replaces events.jsonl")
- Tighten flag parsing boilerplate

Estimated: -34 lines.

### R4: Verify patterns.md templates match v3.0.0 conventions

Run through each template in patterns.md and verify:
- Uses `NN-proposal.md` not `NN-contract.md`
- Uses `pending`/`done` not `not_started`/`in_progress`
- Uses `draft`/`reviewed`/`accepted` not `functional`/`polished`
- No references to removed artifacts (init.md, summary.md, decomposition.md, events.jsonl)
- config.json schema doesn't include `use_codex`

This is the integrity sweep — may find 0 issues or several.
